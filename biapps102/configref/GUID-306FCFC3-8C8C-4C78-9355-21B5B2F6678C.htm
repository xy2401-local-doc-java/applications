HR Supervisor Hierarchy-Based Data Security

Data can be secured via the HR Supervisor Hierarchy using list variable[s], with associated data roles and security filter[s], which are applied at the physical SQL level as IN (a, b, c,...) statement.

Duty Role Mapping

Each duty role grants access to one or more subject areas, and is a member of at least one data security role. You need to map a source role/responsibility to one or more duty roles. See How to Define New Groups and Mappings for Users and BI Roles.
