Exporting Content from the 11.1.1.10.1 ODI Repository

Use these instructions to export content from the 11.1.1.10.1 ODI repository.

To export content from the 11.1.1.10.1 ODI repository:
  1. Launch the ODI Studio client and connect to the ODI repository for Oracle BI Applications 11.1.1.10.1. (Do not connect to the backup of the ODI repository).
  2. Navigate to the Topology tab. From the Connect Navigator (Topology icon drop-down on the top right side of the navigator pane), select Export.

    As part of the procedures described here, you will export files to a local directory.


    Description of the illustration GUID-DA8F74ED-0ED0-4BB7-AEE2-6E71BD7F878D-default.gif
  3. To export connections, complete the following steps:
    1. Launch the Smart Export wizard from the Export selection dialog.

      Description of the illustration GUID-08FF568A-7A03-4CE1-9F4E-2F2993BF72C6-default.gif
    2. Drag the Global context into the Objects to be Exported window. Provide a meaningful name for the export file and click Export.
      This exports the logical and physical topology including assigned Datasource Num ID values and database connect details.

      Description of the illustration GUID-8002AF1A-CF52-46DF-BC9F-9D131C0006C2-default.gif
  4. To export security details, complete the following steps:
    1. Select the option to export security settings.

      Description of the illustration GUID-6ADBD65F-29E0-4C3E-B57B-AE16722D48B7-default.gif
    2. Choose to export to a local file (directory).
      This action exports your user configurations.
  5. To export any customizations in your ODI repository, complete the following steps:
    1. Export the Custom folder by right-clicking it and selecting the Export option.

      Per the customization methodology, all custom and customized ETL tasks are in a separate CUSTOM folder.


      Description of the illustration GUID-E7A2682D-D498-469F-B0FB-321941E7D4C1-default.gif
    2. In the next window, ensure that the Child Components Export check box is selected.

      Description of the illustration GUID-CB930C99-E0BD-47F6-AA2D-507BC6C751F1-default.gif
  6. To export customized data stores, complete the following:
    1. Click the Export option from the navigator window.

      Description of the illustration GUID-C9B4C677-5BD3-496F-A818-825E24B0F92E-default.gif
    2. Select the option to export multiple objects.

      Description of the illustration GUID-7E92DDF8-E620-45CF-BB30-54299A333FCE-default.gif
    3. Drag and drop your purely custom and customized data stores. Ensure that the Export child objects option is selected.
      In the example below, WC_ALLOC_INV_BALANCES_F/FS are purely custom tables while W_GL_OTHER_F/FS are tables that have been customized.

      Description of the illustration GUID-96D93AA1-58A4-4251-BF14-BAD2F62F2503-default.gif
