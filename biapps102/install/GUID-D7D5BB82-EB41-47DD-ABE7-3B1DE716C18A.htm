Creating an Account on the Oracle Service Cloud Site

Use these instructions to create an account with appropriate permissions on the Oracle Service Cloud site.

To create an account on the Oracle Service Cloud site:
  1. Log into the Oracle Service Cloud site using the URL that you used to install the Oracle Service Cloud client.
  2. Create a new profile.
    1. Open the Profile editor (Click Navigation set, click Configuration (Wrench), click Staff Management and then select Profiles).

      Description of the illustration GUID-623F0698-6599-4125-9BEE-8ADF60A2FB31-default.gif
    2. Add a new profile (or use an existing profile).

      Description of the illustration GUID-C1F9502E-D4C1-48B9-9E76-58439BB186F3-default.gif

    3. Provide required details for a new profile.

    4. Click Interfaces menu option.

      Description of the illustration GUID-350929E9-A2C2-4426-A273-DBF7E3A33CCE-default.gif

    5. Provide name and label for each interface.

    6. Click Permissions menu option.

      Description of the illustration GUID-965676DF-03F7-48AD-8F77-812DB2A6E149-default.gif

    7. On the Administration tab, select the Select All check box in the Public SOAP API section.

    8. Navigate to the Organizations tab in the profile and select the Read button to select all Read check boxes.

      Description of the illustration GUID-2EB824F7-5F47-450F-8F2E-6401168F0497-default.gif
    9. Navigate to the Contacts tab in the profile and select the Read button to select all Read check boxes.

      Description of the illustration GUID-07459156-5D49-49D6-A2D5-7D7709A76337-default.gif
    10. Navigate to the Service tab in the profile and select the Read check box in the Incidents area.

      Description of the illustration GUID-7028EFB3-8962-4D25-B037-0CC14A93BFCF-default.gif

    11. Save and close the profile.

  3. Create a Staff Account using the above profile.
    1. Open the Staff Accounts By Group section. (Click Navigation set, click Configuration (Wrench) , click Staff Management and then select Staff Accounts by Group).

      Description of the illustration GUID-A01CD9D9-4E78-4C25-8B1C-8BF54D950949-default.gif
    2. Select New to add a new account.

      Description of the illustration GUID-1502E388-BABD-425F-B0FA-9029838684CC-default.gif

    3. Provide required details (remember the User Name you provide here).

      Note:

      The user name should be something like BIAPPS_INTEG_USER_XXXXXXX where XXXXXXX is a random alpha-numeric string to ensure that integration users configured for an adapter on different sites are unique and the risk of two sites having the same user name is minimized.

    4. In the Profile field, select the Profile created in step 2 above.

    5. In the Group field, either select an existing group or create a new one.

    6. Click the change password button and set a password according to the password rules setup on the site. Remember this password. The user name from earlier step and this password will be needed when registering the Oracle Service Cloud source in Configuration Manager.

    7. Deselect the Password Expires check box.

    8. Save and close the account.

      Description of the illustration GUID-6696E758-D8B7-47A0-AC83-6C1E6080B85E-default.gif

      At this point, the proper account with appropriate permissions has been created on the site. You will need the user name and password that you configured for the user above while registering the Oracle Service Cloud source in Configuration Manager.
