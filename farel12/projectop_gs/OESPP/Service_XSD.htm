Service XSD

A service XSD schema has the following basic structure:

  • Namespace declarations

  • Imports and includes the specifications of XSDs on which this schema definition has dependencies

  • Definition of the elements of the request and response payloads for synchronous and asynchronous versions of all the service operations

    Note: The element names start with the operation names. The suffix determines whether the payload is a request or a response, and whether the payload elements are synchronous or asynchronous.

The following is an excerpt from the SalesLeadService.xsd.
<?xml version="1.0" encoding="UTF-8" ?>
